# Docs

Short summary of the documentation in this directory:

- [`usage.md`](usage.md): canonical consumer guide and public integration contract for
  installed-CLI users of `release-tools`
- [`release-procedures.md`](release-procedures.md): scenario-based procedures
  for binary, container image, and Helm chart releases
- [`self-release.md`](self-release.md): maintainer self-release procedure for
  this repository
- [`agent-release-flow.md`](agent-release-flow.md): implementation rationale, invariants, and reusable
  release-flow pattern for maintainers and agents
- [`cosign-installation.md`](cosign-installation.md): verified Cosign install
  examples for signing-related release flows
- [`future-work.md`](future-work.md): deferred ideas and intentionally
  out-of-scope release-tooling directions

Start with `release-procedures.md` for scenario-based setup steps, then use
`usage.md` as the canonical variable contract.
